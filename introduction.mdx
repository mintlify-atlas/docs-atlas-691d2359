irtrafficgen is a command-line tool that generates HTTPS traffic toward Iran-related internet targets. It samples domains and IP addresses from embedded V2Ray-format `geosite.dat` and `geoip.dat` datasets, then issues HTTP operations under configurable rate limits. This page explains what irtrafficgen does, what it is designed for, and how to navigate the rest of the documentation.

<Warning>
  **Authorized use only.** irtrafficgen can send real HTTPS (and optional DNS) traffic to hosts and IP ranges on the internet. Run it only on networks and targets you own, have written authorization for, or fully control in a lab. The `--live` flag sends real network traffic and is entirely your responsibility. Misuse may violate law or contracts.
</Warning>

## What irtrafficgen does

irtrafficgen embeds `geosite.dat` and `geoip.dat` from the V2Ray project directly in the binary. At startup it loads these datasets and builds a weighted target sampler. Each worker tick picks a target — either a domain from a geosite list or a public IP from `geoip:ir` CIDRs — then issues an HTTPS operation under global QPS limits.

By default the tool runs in **dry-run mode**, which logs planned operations without making any real network connections. Pass `--live` to send real traffic.

## How targets are selected

| Source | When used |
|--------|-----------|
| **Geosite lists** | Domain names from embedded lists. If `geosite_lists` is omitted, every list matching `category-*-ir` is used automatically. |
| **`geoip:ir`** | Random public IPv4/IPv6 from embedded `geoip:ir` CIDRs, subject to `safety.deny_private_ips`. |

Each worker tick picks a target, then picks an operation — `HEAD`, `GET`, or `POST` — according to configured weights when asymmetric mode is enabled. Without asymmetric mode, all operations are `HEAD`.
