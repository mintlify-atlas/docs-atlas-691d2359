---
title: "Install irtrafficgen: from source, Docker, or go build"
description: "Three ways to install irtrafficgen: build from source with the Makefile, run a Docker container, or compile directly with go build and ldflags."
---

irtrafficgen ships as a single self-contained binary with embedded geo datasets — no external files are needed at runtime. You can build it from source using the provided Makefile, run it as a Docker container, or compile it manually with `go build`. All three methods produce a binary with the same functionality.

## Prerequisites

| Requirement | Version | Notes |
|-------------|---------|-------|
| Go | 1.25+ | Required for source and manual builds |
| Git LFS | any | Required if `.dat` files are stored in Git LFS |
| Docker | any | Required for Docker installation only |

The `internal/assets/geosite.dat` and `internal/assets/geoip.dat` files may be tracked with [Git LFS](https://git-lfs.com/). If those files are empty stubs after cloning, run:

```bash
git lfs install
git lfs pull
```

<Tabs>
  <Tab title="Build from source">
    The Makefile handles asset verification and sets version/commit ldflags automatically.

    **Clone and build:**

    ```bash
    git clone https://github.com/arastu/irtrafficgen.git
    cd irtrafficgen
    make check-assets build
    ```

    The `check-assets` target verifies that `geosite.dat` and `geoip.dat` are non-empty before compiling. The resulting `irtrafficgen` binary is placed in the repository root.

    **Build with a version tag:**

    ```bash
    VERSION=1.0.0 make build
    ```

    The Makefile injects the following ldflags:

    ```
    -s -w
    -X github.com/arastu/irtrafficgen/internal/version.Version=$(VERSION)
    -X github.com/arastu/irtrafficgen/internal/version.Commit=$(COMMIT)
    ```

    `COMMIT` is set automatically from `git rev-parse --short HEAD`. If git is not available, it falls back to `none`.

    **Verify the build:**

    ```bash
    ./irtrafficgen version
    ./irtrafficgen inspect
    ```

    **Run tests:**

    ```bash
    make test
    # or: go test ./...
    ```
  </Tab>

  <Tab title="Docker">
    The provided `Dockerfile` uses a two-stage build: a `golang:1.25-alpine` builder stage compiles the binary, and an `alpine:3.21` runtime stage produces a minimal image.

    **Build the image:**

    ```bash
    docker build -t irtrafficgen:latest .
    ```

    **Run in dry-run mode** (no real traffic):

    ```bash
    docker run --rm \
      -v "$(pwd)/config.example.yaml:/etc/irtrafficgen/config.yaml" \
      irtrafficgen:latest \
      run --config /etc/irtrafficgen/config.yaml
    ```

    **Run with live traffic** (authorized environments only):

    ```bash
    docker run --rm \
      -v "$(pwd)/config.example.yaml:/etc/irtrafficgen/config.yaml" \
      irtrafficgen:latest
    ```

    <Note>
      The default `ENTRYPOINT` in the Dockerfile is `["/irtrafficgen", "run", "--config", "/etc/irtrafficgen/config.yaml", "--live"]`. Running the container without arguments sends real traffic. Always mount a config file at `/etc/irtrafficgen/config.yaml`.
    </Note>

    <Warning>
      The Docker entrypoint includes `--live` by default. Mount a config with `dry_run: true` or override the entrypoint if you do not intend to send real traffic.
    </Warning>

    **Inspect embedded data inside the container:**

    ```bash
    docker run --rm irtrafficgen:latest inspect
    ```
  </Tab>

  <Tab title="Manual build">
    You can compile irtrafficgen directly with `go build` without the Makefile. This is useful in CI environments or when you want full control over build flags.

    **Basic build:**

    ```bash
    go build -o irtrafficgen .
    ```

    **Build with version and commit information:**

    ```bash
    go build \
      -trimpath \
      -ldflags="-s -w \
        -X github.com/arastu/irtrafficgen/internal/version.Version=1.0.0 \
        -X github.com/arastu/irtrafficgen/internal/version.Commit=$(git rev-parse --short HEAD)" \
      -o irtrafficgen .
    ```

    **Cross-compile for Linux amd64 from macOS:**

    ```bash
    GOOS=linux GOARCH=amd64 go build \
      -trimpath \
      -ldflags="-s -w" \
      -o irtrafficgen-linux-amd64 .
    ```

    After building, verify the binary and embedded data:

    ```bash
    ./irtrafficgen version
    ./irtrafficgen inspect
    go test ./... -run TestDefaultValidateEmbedded
    ```
  </Tab>
</Tabs>

## Updating embedded geo data

The `geosite.dat` and `geoip.dat` files are embedded at compile time from `internal/assets/`. To update the datasets, replace the files under that directory and rebuild:

```bash
# Replace internal/assets/geosite.dat and internal/assets/geoip.dat, then:
make check-assets build
./irtrafficgen inspect
go test ./... -run TestDefaultValidateEmbedded
```
