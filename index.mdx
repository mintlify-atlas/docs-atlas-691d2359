irtrafficgen is a command-line tool for generating HTTPS traffic toward Iran-related internet targets. It samples domains and IP addresses from embedded V2Ray-format `geosite.dat` and `geoip.dat` datasets, then issues HTTP operations under configurable rate limits. Use it in authorized lab environments to simulate realistic traffic patterns to Iranian internet infrastructure.

<Warning>
  **Authorized use only.** Run irtrafficgen only on networks and targets you own, have written authorization for, or fully control in a lab. The `--live` flag sends real network traffic and is entirely your responsibility.
</Warning>

## How it works

irtrafficgen embeds `geosite.dat` and `geoip.dat` from the V2Ray project. At startup it loads these datasets and builds a weighted target sampler. Each worker tick picks a target — either a domain from a geosite list or a public IP from `geoip:ir` CIDRs — then issues an HTTPS operation.

## Quick example

```bash
# Build the binary
make check-assets build

# Inspect embedded geo datasets
./irtrafficgen inspect

# Run in dry-run mode (no real network I/O)
./irtrafficgen run --config config.example.yaml

# Run with real HTTPS traffic (authorized environments only)
./irtrafficgen run --config config.example.yaml --live
```

<Steps>
  <Step title="Build the binary">
    Run `make check-assets build` to verify that embedded `.dat` files are present and compile the binary.
  </Step>
  <Step title="Inspect geo data">
    Run `./irtrafficgen inspect` to verify that `geosite.dat` and `geoip.dat` loaded correctly and see the available Iran-related lists.
  </Step>
  <Step title="Configure your run">
    Copy `config.example.yaml` and adjust `limits.global_qps`, `max_in_flight`, and target weights for your lab environment.
  </Step>
  <Step title="Send traffic">
    Run with `--live` to send real HTTPS requests. Add `--verbose` to print a session metrics summary when the run stops.
  </Step>
</Steps>
