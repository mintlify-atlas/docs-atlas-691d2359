---
title: "Get started with irtrafficgen in under five minutes"
description: "Build irtrafficgen from source, inspect embedded geo datasets, and send your first dry-run traffic in under five minutes on any Go 1.25+ machine."
---

This guide walks you through building irtrafficgen from source, verifying the embedded geo datasets, and running your first traffic session. The entire flow uses the commands from the README, starting in dry-run mode so no real network traffic is sent until you explicitly opt in.

<Warning>
  **Authorized use only.** The `--live` flag sends real HTTPS traffic to Iran-related targets. Use it only on networks and targets you own or have written authorization for.
</Warning>

<Steps>
  <Step title="Prerequisites">
    You need Go 1.25 or later and Git with LFS support.

    The `.dat` files under `internal/assets/` may be stored in [Git LFS](https://git-lfs.com/). If the files are stubs (zero bytes), install Git LFS and pull them before building:

    ```bash
    git lfs install
    git lfs pull
    ```

    Verify your Go version:

    ```bash
    go version
    # go version go1.25.x linux/amd64
    ```
  </Step>

  <Step title="Clone and build">
    Clone the repository and build the binary using the Makefile. The `check-assets` target confirms that `geosite.dat` and `geoip.dat` are non-empty before compiling:

    ```bash
    git clone https://github.com/arastu/irtrafficgen.git
    cd irtrafficgen
    make check-assets build
    ```

    This produces an `irtrafficgen` binary in the current directory. To embed a version string:

    ```bash
    VERSION=1.0.0 make build
    ```
  </Step>

  <Step title="Inspect embedded data">
    Before running any traffic, verify that the embedded geo datasets loaded correctly:

    ```bash
    ./irtrafficgen inspect
    ```

    The `inspect` command prints each geosite list code, its rule count, and `geoip:ir` CIDR statistics. It exits with code `1` if `geoip:ir` is missing from the embedded data.

    You can also run the embedded data test:

    ```bash
    go test ./... -run TestDefaultValidateEmbedded
    ```
  </Step>

  <Step title="Run in dry-run mode">
    The example config has `dry_run: true`, so no DNS lookups or HTTPS requests are made. This is the safest way to verify your configuration:

    ```bash
    ./irtrafficgen run --config config.example.yaml
    ```

    You should see log lines describing the operations that *would* be sent, including the sampled target and operation type. Press `Ctrl+C` to stop.

    Add `--verbose` to print a session metrics summary on exit:

    ```bash
    ./irtrafficgen run --config config.example.yaml --verbose
    ```

    <Tip>
      `run` is the default subcommand, so `./irtrafficgen --config config.example.yaml` is equivalent to the command above.
    </Tip>
  </Step>

  <Step title="Run with real traffic">
    Once you have confirmed the configuration and have authorization to send traffic, add `--live` to enable real HTTPS connections. The `--live` flag overrides `dry_run: true` in the config file:

    ```bash
    ./irtrafficgen run --config config.example.yaml --live
    ```

    <Warning>
      `--live` sends real HTTPS (and optional DNS) traffic to Iran-related targets. Run this only with explicit authorization. The `--live` flag is entirely your responsibility.
    </Warning>

    To send a single request and exit:

    ```bash
    ./irtrafficgen run --config config.example.yaml --live --once --verbose
    ```
  </Step>
</Steps>
